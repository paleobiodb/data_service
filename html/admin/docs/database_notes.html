authorities top

Records who originally named each taxa and the year of naming. It also records the reference that this naming came from - either the taxa was directly named in a particular reference, or the reference said that the taxa was named earlier by a certain person.

taxon_name Name of this taxon (of type taxon_rank). Note, in this case, if the taxon_rank is species, then it will be a two word species such as "Homo sapiens." Otherwise, it's just one word.

occurrences top

Holds one record for each distinct taxonomic occurrence within a particular collection (usually). Access levels apply to these records

authorizer_no

enterer_no

reference_no

genus_reso

genus_name First part of the name, not necessarily a genus! Could be a higher taxon name.

species_reso

species_name Second part of the name, not necessarily a species! For example, if genus_name is a higher taxon, then species would be "indet."

abund_value Abundance of this occurrence in the collection

abund_unit Units for the abund_value: may be the number of specimens, % of area covered, etc. - several different options.

opinions top

Records information about taxa that have been renamed. Note, don't confuse this with the reidentifications table which is used for reidentifications of a particular fossil at a particular collection.

authorizer_no

enterer_no

modifier_no

opinion_no

reference_no The reference where the opinion came from.

parent_no Key to the authorities table, parent of the child. The child is the original, and the parent_no is the recombination/synonym/etc. according to the status field. Possible values include 'belongs to', 'recombined as', 'subjective synonym of', 'objective synonym of', 'homonym of', 'replaced by', 'corrected as', 'nomen dubium', 'nomen nudum', 'nomen oblitem', 'nomen vanum', 'revalidated'

child_no The child_no field always points to the original taxon_no in the authorities table.

status What does the opinion mean, for example, is it a recombination, synonym, etc.?

diagnosis

ref_has_opinion Basically means that the opinion comes from a refs table entry. If this exists and is equal to 'YES', then go to the refs table to get the publication year (use the reference_no field).

pubyr Year of the opinion. If blank, then look at ref_has_opinion and get the year from the reference instead.
