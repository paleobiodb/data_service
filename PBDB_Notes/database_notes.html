authorities

Records who originally named each taxa and the year of naming. It also records the reference that this naming came from - either the taxa was directly named in a particular reference, or the reference said that the taxa was named earlier by a certain person.

ref_is_authority

taxon_no Can use this to refer back to the opinions table by looking at the parent_no, and child_no fields.

taxon_name Name of this taxon (of type taxon_rank). Note, in this case, if the taxon_rank is species, then it will be a two word species such as "Homo sapiens." Otherwise, it's just one word.

taxon_rank class, species, genus, etc.

pub_yr Year the taxon was named.

type_taxon_no Optional field - for example, Felidae has the type taxon felis. So this is a reference to another taxon_no in the authorities table.

collections

Holds one record for each collection in the database. A collection can be thought of as a particular hole in the ground. Access levels apply to these records

collection_name

collection_aka Another name for the collection. The collection search script looks at both the collection_name and collection_aka when performing a search.

collection_no

reference_no

access_level public, database members, group members, authorizer only. If it's null, which might happen for some of our earliest records, then treat it as public.

release_date After this date, the data is fair game for everyone. Before it, look at the access_level field to determine who should have access.

research_group

correlations

Correlates the scales table with the intervals table.

correlation_no

max_interval_no oldest interval covered by a particular scale.

min_interval_no youngest interval covered by a particular scale.

ecotaph

Attributes of taxa, what they look like, body mass, ocean/land, etc.

ecotaph_no

fivepct

Holds a sample containing 5% of the data in the GeoRef database which is a database of references. Not important to the rest of the database.

images

Records paths to images of occurrences which have been entered into the database. Not much in here yet.

intervals

Each time interval (ie, Cenozoic, etc.) may have an entry in this table. The correlations table lists the relative orderings of the intervals, and the scales table lists opinions about time scales.

interval_no

interval_name

marinepct

Holds references for the 1% study - similar to the fivepct table.. Not important to the rest of the database.

occurrences

Holds one record for each distinct taxonomic occurrence within a particular collection (usually). Access levels apply to these records

occurrence_no

reference_no

collection_no

genus_reso

genus_name First part of the name, not necessarily a genus! Could be a higher taxon name.

species_reso

species_name Second part of the name, not necessarily a species! For example, if genus_name is a higher taxon, then species would be "indet."

abund_value Abundance of this occurrence in the collection

abund_unit Units for the abund_value: may be the number of specimens, % of area covered, etc. - several different options.

opinions

Records information about taxa that have been renamed. Note, don't confuse this with the reidentifications table which is used for reidentifications of a particular fossil at a particular collection.

opinion_no

reference_no The reference where the opinion came from.

parent_no Key to the authorities table, parent of the child. The child is the original, and the parent_no is the recombination/synonym/etc. according to the status field. Possible values include 'belongs to', 'recombined as', 'subjective synonym of', 'objective synonym of', 'homonym of', 'replaced by', 'corrected as', 'nomen dubium', 'nomen nudum', 'nomen oblitem', 'nomen vanum', 'revalidated'

child_no The child_no field always points to the original taxon_no in the authorities table.

status What does the opinion mean, for example, is it a recombination, synonym, etc.?

diagnosis

ref_has_opinion Basically means that the opinion comes from a refs table entry. If this exists, then go to the refs table to get the publication year (use the reference_no field).

pubyr Year of the opinion. If blank, then look at ref_has_opinion and get the year from the reference instead.

refs

Holds one record for each reference (ie, paper).

reference_no

reftitle Title of the reference (ie, paper title).

pubtitle Title of the publication (ie, magazine name).

pubyr Year the reference was published.

author1last Last name of the first author.

author2last Last name of the second author

otherauthors All other authors.

reidentifications

Lists reidentifications of occurences and which reference number reidentified them. For example, this would occur if a fossil was incorrectly classified and later re-classified as something else. Access levels apply to these records

reid_no

reference_no Reference number which reidentified the occurrence.

occurrence_no Occurrence number which was reidentified.

genus_name Newly reidentified genus for this occurrence.

species_name Newly reidentified species for this occurrence.

scales

Listing of time scales. Each one will have been defined by a certain reference. See also the intevals and correlations tables.

scale_no

scale_name

reference_no

secondary_refs

Allows people to enter more than one reference for a collection. Basically just ties a collection number to a reference number, so if you wanted to have several references, you'd have an entry in here for each refernce (not including the primary reference).

collection_no

reference_no

session_data

Holds session id, current enterer, authorizer, etc. which we store in cookies.

queue Holds list of parameters from the user's form submission(?).
